## `lms-import.png` is captured from SCORM Cloud, not locally

A real-LMS import screenshot requires an authenticated LMS session, which can't be produced in a
local/headless toolkit run. Capture it from the SCORM Cloud conformance run that already exercises
these packages (`.github/workflows/scorm-cloud.yml`) — import the walkthrough `scorm12` package,
play it to completion, and screenshot the registration showing COMPLETED/PASSED. Add the file here
and drop the "_Pending_" note. Until then the website how-it-works page should use `harness.png` and
either omit the LMS shot or mark it coming-soon — **do not** mock one up (honesty constraint).
